OSS源文件配置Header头部Content-Type

Posted on 2018-10-19 | Edited on 2019-06-24 | In OSS | Comments:

网站的一些静态资源存放在阿里云的OSS上，通过CDN加速。有个小项目存放了几个音频，希望上传后能通过在线试听确认资源可用。之前是直接在CDN上设置HTTP响应头Content-Type，但是这个只能设置一个值。
这里记录下提交工单工程师给的解决方法。
HTTP响应头Content-Type参数一般都是在源站(OSS)中设置，CDN一般都是不设置的。

由于ossutil不支持同步(即远程目录与本地目录一致)，我是通过ossfs把OSS挂载到ECS上，本地通过rclone的sftp同步到oss。默认上传到OSS的文件Content-Type全是”application/octet-stream”。
ossfs通过查询/etc/mime.types中的内容来确定文件的Content-Type，检查文件是否存在，不存在通过以下方法添加：

  • 对于Ubuntu可通过sudo apt-get install mime-support来添加；
  • 对于Centos可通过sudo yum -y install mailcap来添加；
  • 也可手动添加，每种格式一行，格式为：video/mp4 mp4 mpg4 m4v

添加完/etc/mime.types后，重新挂载oss，后面上传的文件就自动识别文件类型了。
